# Инструкция: Создание карточки объекта из каталога новостроек

## Обзор

Эта инструкция описывает, как создать карточку объекта (проекта) для каталога новостроек, аналогичную существующей реализации в проекте.

## 1. Структура данных объекта

Каждый объект должен содержать следующие поля:

```javascript
{
  id: 1,                              // Уникальный идентификатор
  name: 'ЖК Премиум Парк',            // Название проекта
  district: 'Ленинский район',        // Район расположения
  type: 'Монолитно-кирпичный',       // Тип строительства (опционально)
  description: 'Краткое описание...', // Краткое описание (2-3 строки)
  fullDescription: 'Полное описание...', // Полное описание для страницы деталей
  price: '350 000 ₽',                 // Цена за м² (опционально)
  priceFrom: '8 500 000 ₽',           // Цена от (обязательно)
  completion: 'Q2 2025',              // Срок сдачи
  rooms: '1-4 комн.',                 // Количество комнат
  parking: 'Подземный',               // Тип парковки (опционально)
  status: 'Строятся',                 // Статус проекта (Строятся, Сданные, Старт продаж)
  discount: '-10%',                   // Скидка (опционально)
  image: 'URL изображения',           // Основное изображение (fallback)
  images: [                           // Массив изображений (приоритет)
    'URL изображения 1',
    'URL изображения 2',
    'URL изображения 3'
  ],
  developer: 'ООО "Премиум Строй"',   // Застройщик (опционально)
  floors: '15-25 этажей',             // Этажность (опционально)
  apartments: '450 квартир',          // Количество квартир (опционально)
  area: 'от 35 до 120 м²',            // Площадь квартир
  features: [                          // Особенности комплекса (опционально)
    'Детская площадка',
    'Спортивная зона',
    'Парковка'
  ]
}
```

## 2. Компонент ProjectCard

### Расположение файла
`src/components/ProjectCard.jsx`

### Основные возможности компонента:

1. **Слайдер изображений**
   - Поддержка нескольких изображений
   - Навигация стрелками (← →)
   - Навигация точками (индикаторы)
   - Поддержка свайпов на мобильных устройствах
   - Автоматическое переключение изображений

2. **Адаптивный дизайн**
   - Мобильная версия: изображение сверху, информация снизу
   - Десктопная версия: изображение справа (40% ширины), информация слева (60% ширины)

3. **Отображение информации**
   - Название проекта и район
   - Краткое описание (до 2 строк)
   - Цена от (с форматированием)
   - Срок сдачи
   - Количество комнат
   - Площадь
   - Статус проекта (бейдж)
   - Скидка (бейдж, если есть)

4. **Навигация**
   - Кнопка "Подробнее" ведет на страницу деталей проекта (`/project/:id`)

## 3. Использование компонента

### Импорт компонента

```javascript
import ProjectCard from '../components/ProjectCard'
```

### Рендеринг карточки

```javascript
// Одиночная карточка
<ProjectCard project={projectData} />

// Список карточек
{projects.map((project) => (
  <ProjectCard key={project.id} project={project} />
))}
```

### Пример использования в HomePage

```javascript
import ProjectCard from '../components/ProjectCard'

// В компоненте
{filteredProjects.length > 0 ? (
  <div className="space-y-4">
    {filteredProjects.map((project) => (
      <ProjectCard key={project.id} project={project} />
    ))}
  </div>
) : (
  <div className="text-center py-12">
    <p className="text-gray-600 text-lg">По заданным фильтрам ничего не найдено</p>
  </div>
)}
```

## 4. Зависимости

### React Router
Компонент использует `useNavigate` из `react-router-dom` для навигации:

```javascript
import { useNavigate } from 'react-router-dom'
```

### Tailwind CSS
Компонент использует классы Tailwind CSS для стилизации. Убедитесь, что Tailwind настроен в проекте.

## 5. Ключевые функции компонента

### Форматирование цены
```javascript
const formatPrice = (price) => {
  // Форматирует цену с пробелами между каждыми тремя цифрами
  // Пример: "8500000" → "8 500 000 ₽"
}
```

### Обработка изображений
- Поддерживает массив изображений (`images`)
- Поддерживает одно изображение (`image`)
- Поддерживает JSON-строку с изображениями
- Автоматически использует первое изображение как основное

### Навигация по изображениям
- Клавиатурная навигация (стрелки ← →)
- Кнопки навигации на изображении
- Индикаторы (точки) внизу изображения
- Свайпы на мобильных устройствах

## 6. Стили и классы

### Основные классы Tailwind:
- `bg-white` - белый фон
- `rounded-lg` - скругленные углы
- `shadow-md` - тень
- `hover:shadow-lg` - тень при наведении
- `transition-shadow` - плавный переход тени
- `border border-gray-200` - серая рамка

### Цветовая схема:
- `primary-600` - основной цвет (кнопки, бейджи)
- `primary-700` - темнее основной цвет (hover)
- `gray-800` - темно-серый (текст)
- `gray-600` - серый (вторичный текст)
- `gray-500` - светло-серый (метки)

## 8. Интеграция с API

Если данные приходят с сервера, убедитесь, что структура данных соответствует ожидаемой:

```javascript
// В services/api.js
export const getProjects = async (filters = {}) => {
  const response = await fetch(`${API_URL}/projects`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(filters)
  })
  return response.json()
}
```

## 12. Troubleshooting

### Изображения не отображаются
- Проверьте формат данных: массив или строка
- Убедитесь, что URL изображений корректны
- Проверьте наличие поля `image` как fallback

### Навигация не работает
- Убедитесь, что `react-router-dom` установлен
- Проверьте настройку роутера в приложении

### Стили не применяются
- Убедитесь, что Tailwind CSS настроен
- Проверьте наличие классов в конфигурации Tailwind
